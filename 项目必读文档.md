## 一、数据存储与更新机制

### 1.1 缓存文件位置

所有缓存数据位于 `data/` 目录下：

| 文件 | 格式 | 内容 | 更新策略 |
|------|------|------|---------|
| `klines.parquet` | Parquet | 10年日线行情（open/high/low/close/volume/amount/turnover/pe/pb） | 24小时过期，增量追加 |
| `fundamentals.parquet` | Parquet | 近一期财报数据（ROE/净利率/负债率/现金流等） | 7天过期，按需刷新 |
| `csi500_constituents.csv` | CSV | 最新500只成分股列表 | 每次实时拉取，失败回退缓存 |

源码路径定义在 `config.py`：
```python
KLINE_CACHE_FILE = DATA_DIR / "klines.parquet"
FUNDA_CACHE_FILE = DATA_DIR / "fundamentals.parquet"
```

### 1.2 是否每日自动更新

**是，但采用增量模式，不是实时流式。**

每次运行 `main.py` 时，`data_fetcher.fetch_daily_klines()` 自动执行：

```
有缓存 → 只拉缓存最新日期之后的缺失数据 → 增量合并 → 去重保存
无缓存 → 全量拉取10年历史
```

效果：
- 开盘日第一次跑 → 自动补充到最新交易日
- 同一日重复跑 → 增量更新为 0 条
- 基本面按财报期获取，7天内不重复拉取

## 二、模块架构与调用接口

### 2.1 管线总览

```
main.py (入口)
  │
  ├─ Step 1: fetch_csi500_constituents()          → 成分股列表
  ├─ Step 2: fetch_daily_klines(symbols)           → 日线行情 (含PE/PB)
  ├─ Step 3: fetch_fundamentals(symbols)           → 财务数据
  │          calculate_fundamental_scores(funda)   → 基本面评分
  ├─ Step 4:
  │   ├─ calculate_valuation_scores(klines)        → 估值评分
  │   ├─ calculate_volume_scores(klines)           → 量能评分
  │   ├─ calculate_capital_flow_scores(klines)     → 量价配合评分
  │   └─ calculate_final_scores(...)               → 综合评分
  ├─ Step 5: generate_report / generate_html_report  → 报告输出
  └─ Step 6: save_report                           → 文件保存(含旧报告清理)
```

### 2.2 各模块独立调用入口

#### 估值分析 `valuation_analyzer.py`

```python
from valuation_analyzer import calculate_valuation_scores

valuation_scores = calculate_valuation_scores(klines)
# 输出列: symbol, pe, pb, pe_rank, pb_rank, close_percentile,
#         pe_score, pb_score, price_score, valuation_score
```

子因子评分公式：
- `pe_score`：PE越低分越高（负PE固定10分）
- `pb_score`：PB越低分越高（负PB固定10分）
- `price_score`：价格在历史 p5-p95 分位数中的位置，低位高分
- `valuation_score` = 0.4×pe_score + 0.3×pb_score + 0.3×price_score

#### 基本面分析 `fundamental_analyzer.py`

```python
from fundamental_analyzer import fetch_fundamentals, calculate_fundamental_scores

# Step 1: 获取原始数据（自动走缓存）
fundamentals = fetch_fundamentals(symbols)

# Step 2: 计算评分
scores = calculate_fundamental_scores(fundamentals)
# 输出: symbol, roe, np_margin, gp_margin, net_profit, eps_ttm,
#       liability_ratio, cfo_to_np, is_loss,
#       roe_score, np_margin_score, debt_score, cashflow_score,
#       fundamental_score
```

子因子评分公式：
- `roe_score`：ROE≥20%→95分，ROE≤0%→10分
- `np_margin_score`：净利率≥20%→90分，≤0%→10分
- `debt_score`：负债率≤20%→90分，≥95%→10分（含金融行业85-95%特判）
- `cashflow_score`：cfo_to_np≥2→90分，cfo_to_np上限5（极值保护）
- `fundamental_score` = 0.35×roe + 0.25×margin + 0.25×debt + 0.15×cashflow
- 亏损惩罚：研发型（现金流为正）×0.6 上限55；真实亏损×0.3 上限40

#### 量能分析 `volume_analyzer.py`

```python
from volume_analyzer import calculate_volume_scores

vol_scores = calculate_volume_scores(klines)
# 输出: symbol, volume_ratio_20, turnover, price_trend_5d,
#       price_trend_20d, volume_trend_5d, price_volume_score,
#       volume_zscore, volume_score
```

子因子评分：
- `price_volume_score`：上涨放量80分 / 上涨正常60分 / 下跌放量20分 / 下跌缩量60分
- `volume_zscore`：成交量Z-score突变检测（>3异常，>2明显放量）
- `volume_score` = 0.35×量比 + 0.30×量价配合 + 0.20×突变 + 0.15×换手率

#### 量价配合 `capital_flow_analyzer.py`

```python
from capital_flow_analyzer import calculate_capital_flow_scores

cf_scores = calculate_capital_flow_scores(klines)
# 输出: symbol, price_trend_5d, vol_trend_5d, vol_ratio_10_60,
#       divergence_score, vol_trend_score, amp_score, capital_flow_score
```

子因子评分：
- `divergence_score`：OHLC位置分析（收盘在振幅上1/3→买入压力60-85分；下1/3→卖出压力15-40分）
- `vol_trend_score`：10日均量/60日均量，放量→活跃高分
- `amp_score`：近20日振幅中位数，活跃度评分
- `capital_flow_score` = 0.40×OHLC压力 + 0.35×量趋势 + 0.25×振幅活跃度

#### 综合评分 `scorer.py`

```python
from scorer import calculate_final_scores

final = calculate_final_scores(
    constituents,          # [symbol, stock_name, weight]
    valuation_scores,
    volume_scores,
    fundamental_scores,
    capital_flow_scores,
)
# 输出: symbol, stock_name, total_score, rank, action, 及各子因子
```

**总分公式**（权重配置在 `config.SCORE_WEIGHTS`）：

| 因子 | 权重 | 数据来源 |
|------|:----:|---------|
| 估值 | 25% | valuation_analyzer |
| 基本面 | 25% | fundamental_analyzer |
| 量能 | 25% | volume_analyzer |
| 动量 | 15% | 40%×price_trend_5d + 60%×price_trend_20d |
| 量价配合 | 10% | capital_flow_analyzer |

**操作建议阈值**（配置在 `config.SCORE_THRESHOLDS`）：
- ≥75：推荐关注
- ≥65：可以关注
- ≥50：持有观望
- ≥35：谨慎观察
- <35：注意风险

## 三、在新对话中快速复用

### 3.1 完整管线（推荐）

```python
import sys
sys.path.insert(0, r"D:\qoder workplace\test 1")

from config import DATA_DIR
from data_fetcher import (
    fetch_csi500_constituents, fetch_daily_klines, fetch_latest_trade_date,
)
from valuation_analyzer import calculate_valuation_scores
from volume_analyzer import calculate_volume_scores
from fundamental_analyzer import fetch_fundamentals, calculate_fundamental_scores
from capital_flow_analyzer import calculate_capital_flow_scores
from scorer import calculate_final_scores

# 1. 获取成分股
constituents = fetch_csi500_constituents()
symbols = constituents["symbol"].tolist()

# 2. 获取日线行情（自动使用缓存+增量更新）
trade_date = fetch_latest_trade_date()
klines = fetch_daily_klines(symbols, end_date=trade_date, years=10)

# 3. 获取基本面（自动使用缓存）
fundamentals = fetch_fundamentals(symbols)

# 4. 计算各维度评分
val_scores = calculate_valuation_scores(klines)
vol_scores = calculate_volume_scores(klines)
funda_scores = calculate_fundamental_scores(fundamentals)
cf_scores = calculate_capital_flow_scores(klines)

# 5. 综合评分
final = calculate_final_scores(
    constituents, val_scores, vol_scores, funda_scores, cf_scores
)

# 6. 查看前10名
print(final.head(10)[["symbol", "stock_name", "total_score", "action"]])
```

## 四、配置文件说明

所有可调参数集中在 `config.py`：

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| `KLINE_CACHE_FILE` | 日线缓存路径 | `data/klines.parquet` |
| `FUNDA_CACHE_FILE` | 基本面缓存路径 | `data/fundamentals.parquet` |
| `CACHE_CONFIG.kline_max_age_hours` | 日线缓存过期时间 | 24小时 |
| `CACHE_CONFIG.funda_max_age_days` | 基本面缓存过期时间 | 7天 |
| `SCORE_WEIGHTS` | 五因子权重 | valuation=0.25, fundamental=0.25, volume=0.25, momentum=0.15, capital_flow=0.10 |
| `SCORE_THRESHOLDS` | 操作建议阈值 | strong_buy=75, buy=65, hold=50, caution=35 |
| `FUNDAMENTAL_CONFIG.scoring` | 基本面子因子权重 | roe=0.35, margin=0.25, debt=0.25, cashflow=0.15 |
| `FUNDAMENTAL_CONFIG.filters.penalize_loss` | 是否启用亏损惩罚 | True |
| `REPORT_CONFIG.top_n` | Top N 推荐数量 | 10 |
| `REPORT_CONFIG.max_txt_reports` | 保留最大文本报告数 | 20 |

## 五、注意事项

1. **数据源**：日线行情来自 **baostock**（免费稳定），成分股来自 **akshare**（中证指数官网）
2. **量价配合说明**：本模块基于公开行情数据（OHLC位置+成交量趋势）间接估算，**并非真实Level-2资金流数据**
3. **亏损股处理**：净利润为负的股票会自动被打上"亏损暂避"标签，评分大幅降低
4. **评分仅为参考**：因子评分用于量化辅助决策，不构成投资建议
